# 本文档中
使用 git 命令进行双人协作开发。本文档用于记录常用的 git 命令，方便后续使用。

# git 操作规范

在双人协作开发中，保持代码库的整洁和同步是非常重要的。以下是一般的开发流程以及各个阶段对应的 Git 操作指南。

## 一、 双人协作一般流程

1. **获取代码**：将远程项目代码克隆到本地。
2. **创建分支**：为了不影响主干代码，每次开发新功能或修复 Bug 时，从主分支拉取并创建一个新的个人分支。
3. **开发与提交**：在个人分支上进行代码编写，并频繁地进行本地提交（Commit）。
4. **同步主干（重要）**：在准备将自己的代码推送到远程之前，先拉取（Pull）主干分支的最新代码到本地，以防同伴已经更新了代码。
5. **解决冲突**：如果拉取时发现两人修改了同一处代码，需要手动解决冲突。
6. **推送与合并**：将本地分支推送（Push）到远程仓库，并通过发起合并请求（Pull Request/Merge Request）将代码合并回主分支。

---

## 二、 各阶段 Git 命令详解与操作指南

### 1. 初始化与获取项目 (Clone)
* **用途**：第一次参与项目时，将远程仓库的代码完整下载到本地电脑。
* **终端命令**：
  ```bash
  git clone <远程仓库地址>
  ```
* **GUI 操作 (以 VS Code 为例)**：
  * 点击左侧活动栏的 **“源代码管理”** 图标（分支形状）。
  * 点击 **“克隆存储库”** (Clone Repository) 按钮。
  * 在顶部弹出的输入框中粘贴远程仓库 URL 并按回车。

### 2. 创建与切换分支 (Branch/Checkout)
* **用途**：创建独立的工作区进行开发，避免两人同时修改主干分支（如 `main` 或 `master`）导致混乱。
* **终端命令**：
  ```bash
  # 创建并立即切换到新分支
  git checkout -b <你的分支名称>
  # 或者使用较新的 switch 命令
  git switch -c <你的分支名称>
  ```
* **GUI 操作**：
  * 点击 VS Code 界面 **左下角** 的当前分支名称（例如 `main`）。
  * 在弹出的菜单中选择 **“创建新分支...”** (Create new branch...)。
  * 输入新的分支名称后按回车。

### 3. 暂存与提交更改 (Add & Commit)
* **用途**：当你完成了一小部分工作时，将修改的文件“打包”并记录到本地的 Git 历史中。
* **终端命令**：
  ```bash
  # 暂存所有更改的文件
  git add .
  
  # 提交更改并添加描述信息
  git commit -m "feat: 添加了登录页面的基础UI"
  ```
* **GUI 操作**：
  * 进入 **“源代码管理”** 面板。
  * 在“更改”列表下，鼠标悬停在文件上，点击右侧的 **`+` 号**（暂存更改）。
  * 在顶部的 **“消息”** 输入框中填写提交说明。
  * 点击下方的 **“提交”** (Commit) 按钮。

### 4. 拉取/同步最新代码 (Pull)
* **用途**：获取同伴推送到远程仓库的最新代码。通常在自己准备提交推送之前，或者每天开始工作时执行。
* **终端命令**：
  ```bash
  # 拉取指定分支的最新代码（通常需要拉取主干最新代码以同步）
  git pull origin <分支名称>
  ```
* **GUI 操作**：
  * 在 **“源代码管理”** 面板，点击右上角的 **`...` (更多操作)** 图标。
  * 在下拉菜单中选择 **“拉取”** (Pull)。
  * 或者直接点击面板中的 **“同步更改”** (Sync Changes) 按钮（这会同时执行 Pull 和 Push）。

### 5. 解决冲突 (Resolve Conflicts)
* **用途**：当两人修改了同一个文件的同一行代码时，Git 无法自动合并，需要人工介入决定保留哪部分代码。
* **终端命令**：
  * 冲突发生后，手动打开冲突文件修改代码。
  * 修改完成后执行：
    ```bash
    git add <冲突的文件>
    git commit -m "fix: 解决合并冲突"
    ```
* **GUI 操作**：
  * VS Code 会在文件树中用 **`!`** 或红色高亮标记有冲突的文件。
  * 打开该文件，编辑器上方会出现几个快捷按钮：**“保留当前更改”** (Accept Current Change)、**“保留传入的更改”** (Accept Incoming Change)、**“保留双方更改”** (Accept Both Changes)。
  * 点击合适的选项（或手动编辑）解决后，保存文件。
  * 在 **“源代码管理”** 面板点击 `+` 号暂存，然后点击 **“提交”**。

### 6. 推送本地分支 (Push)
* **用途**：将你本地已经 Commit 的代码上传到远程服务器，让同伴能够看到你的进度。
* **终端命令**：
  ```bash
  # 第一次推送新分支需要设置上游分支
  git push -u origin <你的分支名称>
  
  # 后续推送只需执行
  git push
  ```
* **GUI 操作**：
  * 如果是新建的分支，在 **“源代码管理”** 面板会出现一个 **“发布 Branch”** (Publish Branch) 按钮，点击即可。
  * 如果是已发布的分支，点击 **“同步更改”** (Sync Changes) 按钮。

### 7. 合并代码 (Merge / Pull Request)
* **用途**：你的功能开发完毕后，需要将你的分支代码合并到主干（`main` 或 `dev`）。在双人协作中，推荐通过提 PR 的方式让同伴进行 Code Review（代码审查）。
* **终端操作 (本地合并)**：
  ```bash
  git checkout main        # 切换回主分支
  git pull origin main     # 确保主分支是最新的
  git merge <你的分支名称>   # 将你的分支合并到主分支
  git push origin main     # 推送合并后的主分支
  ```
* **GUI 操作 (推荐使用网页端)**：
  * 在本地推送到远程后，打开代码托管平台网页（如 GitHub / GitLab / 腾讯工蜂 等）。
  * 页面通常会自动提示你有一个新推送的分支，点击 **"Compare & pull request"**（对比并创建合并请求）。
  * 填写相关说明后，指派给你的同伴进行 Review，同伴确认无误后点击网页上的 **"Merge"** 按钮完成合并。
