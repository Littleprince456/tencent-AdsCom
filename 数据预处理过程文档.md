## 概述

本文档详细描述 TAAC2026 比赛数据集的完整数据预处理流程。该数据集包含约 2 亿条用户序列，采用扁平列布局（flat column layout）的 Parquet 格式存储，共 120 列原始特征。

预处理的目标是将原始数据转换为模型可用的张量格式，包括：
- 数据清洗
- 特征填充与对齐
- 缺失值处理
- 异常值检测与处理
- 时间分桶等特征工程

---

## 原始数据格式

原始数据按照 `示例数据格式.md` 描述，分为 5 大类特征：

| 特征分组       | 列数 | 数据形态                | 说明                                           |
| -------------- | ---- | ----------------------- | ---------------------------------------------- |
| ID 与标签      | 5    | `int64` / `int32`       | 核心标识、监督标签和时间戳                     |
| 用户整型特征   | 46   | `int64` / `list<int64>` | 单值或多值离散用户特征，描述用户属性与偏好     |
| 用户稠密特征   | 10   | `list<float>`           | 连续值用户特征，包含 embedding 与对齐统计信号  |
| 物品整型特征   | 14   | `int64` / `list<int64>` | 离散物品特征，包含类目、类型、基础信息与多标签 |
| 域行为序列特征 | 45   | `list<int64>`           | 来自 4 个行为域的用户行为序列特征              |

原始数据共 120 列，形状为 `(N, 120)`，其中 N 是样本数（约 2 亿条）。

## 数据预处理完整流程

### 整体流程图

```
原始 Parquet 文件 (120 列)
    ↓
[1] 数据加载与 schema 解析
    ↓
[2] 数据清洗
    ├─ null → 0
    ├─ -1 → 0
    └─ <= 0 → 0
    ↓
[3] 标签生成
    └─ label_type == 2 → 正样本 (1)，其他 → 负样本 (0)
    ↓
[4] 特征分组处理
    ├─ 用户整数特征：填充 → 拼接 → 检查越界
    ├─ 物品整数特征：填充 → 拼接 → 检查越界
    ├─ 用户稠密特征：填充 → 拼接
    └─ 序列特征：填充 → 时间分桶 → 形状重构
    ↓
[5] 输出模型张量格式
```

### 预处理详细步骤

| 步骤 | 说明 |
|-----|------|
| **1. 数据加载** | 读取 schema.json，解析特征分组和维度信息；使用 PyArrow 读取 Parquet 文件 |
| **2. 值清理** | 所有整数类特征的特殊值处理：`null`、`-1`、`<=0` 统一映射为 0（填充值） |
| **3. 填充对齐** | 变长特征（列表形式）填充至固定长度 |
| **4. 特征拼接** | 同组特征按扁平格式拼接为一个张量 |
| **5. 越界检查** | 检查特征值是否超出 vocab 范围，可选择裁剪或报错 |
| **6. 时间特征工程** | 序列特征的时间戳进行差分和分桶处理 |
| **7. 标签二值化** | 将多类别标签转换为二分类标签（转化/未转化） |

## 各类特征详细处理

### ID 与标签列

#### 原始特征说明
- **列数**：5 列
- **数据类型**：`int64` / `int32`
- **字段**：`user_id`, `item_id`, `label_type`, `label_time`, `timestamp`
- **特点**：无空值

#### 预处理操作

| 字段 | 预处理操作 | 说明 |
|-----|-----------|------|
| `timestamp` | 直接保留 | 样本发生时间戳 |
| `label_type` | 二值化转换 | `label_type == 2` → 1（正样本），其他 → 0（负样本） |
| `label_time` | 暂不使用 | 标签时间，当前 baseline 未使用 |
| `user_id` | 保留为列表 | 用户唯一标识 |
| `item_id` | 暂不使用 | 物品唯一标识 |

#### 处理示例（实例 1）

**原始数据（单样本）：**
```python
{
    'user_id': 1001,
    'item_id': 2002,
    'label_type': 2,          # 转化样本
    'label_time': 1712345678,
    'timestamp': 1712345680
}
```

**处理后：**
```python
{
    'user_id': 1001,
    'label': 1,               # 正样本
    'timestamp': 1712345680
}
```

#### 处理示例（实例 2）

**原始数据（单样本）：**
```python
{
    'user_id': 1002,
    'item_id': 2003,
    'label_type': 1,          # 非转化样本
    'label_time': 1712345680,
    'timestamp': 1712345685
}
```

**处理后：**
```python
{
    'user_id': 1002,
    'label': 0,               # 负样本
    'timestamp': 1712345685
}
```

#### 形状变化
- 保留 3 个可用字段
- 新增 `label` 字段
- 形状：`(5,)` → 保留并转换为 `(3,)` + 新增 1 个

### 用户整型特征

#### 原始特征说明
- **列数**：46 列
- **命名**：`user_int_feats_{id}`
- **数据类型**：`int64` 标量 或 `list<int64>` 数组
- **包含内容**：用户属性、用户偏好等离散特征

#### 预处理操作

| 操作类型 | 步骤 | 说明 |
|---------|------|------|
| **值清理** | 1 | `null` → 0 |
| | 2 | `-1` → 0 |
| | 3 | `<= 0` → 0 |
| **填充对齐** | 4 | 标量特征：维度 1，直接填入 |
| | 5 | 数组特征：填充至固定长度，长度由 schema.json 指定 |
| **特征拼接** | 6 | 所有特征按顺序拼接为扁平张量 |
| **越界检查** | 7 | 检查是否 >= vocab_size，可选裁剪为 0 或报错 |
| | 8 | vocab_size == 0 的特征强制为 0 |

#### 处理示例（实例 1 - 标量特征）

**原始数据（单样本）：**
```python
user_int_feats_5 = 123
```

**处理过程：**
1. 值清理：`123 > 0` → 保持 `123`
2. 填充：标量，无需填充 → `[123]`
3. 检查越界：假设 vocab_size = 1000，`123 < 1000` → 通过
4. 输出：`[123]`

**最终结果：**
```python
[123]
```

#### 处理示例（实例 2 - 标量特征，含异常值）

**原始数据（单样本）：**
```python
user_int_feats_6 = -1
```

**处理过程：**
1. 值清理：`-1 <= 0` → 替换为 `0`
2. 填充：标量 → `[0]`
3. 输出：`[0]`

**最终结果：**
```python
[0]
```

#### 处理示例（实例 3 - 变长数组特征）

**原始数据（单样本）：**
```python
user_int_feats_62 = [1, 0, 5, -1, 8, 3]  # 长度 6
```
假设 schema 中该特征长度为 10

**处理过程：**
1. 值清理：`[1, 0, 5, 0, 8, 3]`
2. 填充至 10：`[1, 0, 5, 0, 8, 3, 0, 0, 0, 0]`
3. 检查越界：假设 vocab_size = 100，所有值 < 100 → 通过

**最终结果：**
```python
[1, 0, 5, 0, 8, 3, 0, 0, 0, 0]
```

#### 形状变化（示例）
```
原始状态（按列）：
  user_int_feats_1:  [B]
  user_int_feats_2:  [B]
  user_int_feats_3:  ListArray
  ...
  user_int_feats_46: [B]

处理后（扁平化）：
  user_int_feats:    [B, U_total_dim]  int64
```

其中 `U_total_dim = user_int_schema.total_dim`

### 域行为序列特征

#### 原始特征说明
- **列数**：45 列
- **数据类型**：`list<int64>` 数组
- **分域分布**：
  - `domain_a_seq_{38-46}`：9 列
  - `domain_b_seq_{67-79, 88}`：14 列
  - `domain_c_seq_{27-37, 47}`：12 列
  - `domain_d_seq_{17-26}`：10 列

#### 预处理操作

| 操作类型 | 步骤 | 说明 |
|---------|------|------|
| **序列填充** | 1 | 所有序列特征填充至 `max_len`（默认 256） |
| **值清理** | 2 | `null`、`-1`、`<= 0` → 0 |
| **形状重组** | 3 | 从多个 ListArray 重组为 `[B, n_feats, max_len]` 形状 |
| **序列长度** | 4 | 记录每个样本的有效序列长度 |
| **时间分桶** | 5 | 对时间戳特征进行差分和分桶处理 |
| **越界检查** | 6 | 检查特征值范围 |

#### 时间分桶详细说明

**处理步骤：**
1. 读取时间戳序列，填充至 `max_len` → 形状 `[B, max_len]`
2. 计算时间差：`timestamp_expanded - ts_padded` → 取 `max(0, ...)`
3. 使用 `np.searchsorted` 在 `BUCKET_BOUNDARIES` 中查找位置
4. 桶 ID = `searchsorted_result + 1`
5. 填充位置桶 ID = 0

**BUCKET_BOUNDARIES 定义（64 个边界，65 个桶）：**
- 秒级：5, 10, 15, ..., 60（12 个）
- 分钟级：120, 180, ..., 3600（21 个）
- 小时级：5400, ..., 86400（18 个）
- 天级：172800, ..., 604800（7 个）
- 年级：11664000, ..., 31536000（6 个）

#### 处理示例（实例 1 - 序列特征值）

**原始数据（单样本）：**
```python
domain_a_seq_38 = [10, -1, 15, 0, 20]  # item 序列
domain_a_seq_39 = [1, 2, 3, 4, 5]      # action 序列
```
假设 `max_len = 8`，`n_feats = 2`

**处理过程：**
1. 值清理：
   - seq_38 → `[10, 0, 15, 0, 20]`
   - seq_39 → `[1, 2, 3, 4, 5]`
2. 填充至 8：
   - seq_38 → `[10, 0, 15, 0, 20, 0, 0, 0]`
   - seq_39 → `[1, 2, 3, 4, 5, 0, 0, 0]`
3. 形状重组为 `[2, 8]`

**最终结果：**
```python
# domain_a 特征
[
    [10, 0, 15, 0, 20, 0, 0, 0],  # feature 38
    [1, 2, 3, 4, 5, 0, 0, 0]     # feature 39
]
# domain_a_len
5
```

#### 处理示例（实例 2 - 时间分桶）

**原始数据（单样本）：**
```python
timestamp = 1712345680              # 当前样本时间
domain_a_seq_ts = [                 # 历史行为时间戳
    1712345670,
    1712345600,
    1712345000
]
```
假设 `max_len = 5`

**处理过程：**
1. 填充时间戳序列：`[1712345670, 1712345600, 1712345000, 0, 0]`
2. 计算时间差（秒）：
   - `1712345680 - 1712345670 = 10`
   - `1712345680 - 1712345600 = 80`
   - `1712345680 - 1712345000 = 680`
   - 填充位置：0
   → `[10, 80, 680, 0, 0]`
3. 查找桶位置：
   - 10 → 在边界 [5, 10] 后 → index=2 → bucket=3
   - 80 → 在边界 [60, 120] 后 → index=12 → bucket=13
   - 680 → 在边界 [600, 900] 后 → index=22 → bucket=23
   - 0 → bucket=0

**最终结果：**
```python
# domain_a_time_bucket
[3, 13, 23, 0, 0]
```

#### 处理示例（实例 3 - 空序列）

**原始数据（单样本）：**
```python
domain_b_seq_67 = []
domain_b_seq_68 = []
domain_b_seq_ts = []
```

**处理过程：**
1. 全部填充为 0
2. 序列长度 = 0

**最终结果：**
```python
# domain_b 特征 (n_feats=2, max_len=256)
[
    [0, 0, 0, ..., 0],  # 256 个 0
    [0, 0, 0, ..., 0]
]
# domain_b_len
0
# domain_b_time_bucket
[0, 0, 0, ..., 0]
```

#### 形状变化
```
原始状态（按域特征列）：
  domain_a_seq_38:  ListArray
  ...
  domain_a_seq_46:  ListArray

处理后（按域）：
  domain_a:              [B, n_feats_a, max_len_a]  int64
  domain_a_len:          [B]                         int64
  domain_a_time_bucket:  [B, max_len_a]              int64
  
  domain_b:              [B, n_feats_b, max_len_b]  int64
  domain_b_len:          [B]                         int64
  domain_b_time_bucket:  [B, max_len_b]              int64
  
  # domain_c, domain_d 同理
```

## 缺失值与异常值处理

### 缺失值处理策略

| 情况 | 处理方式 | 说明 |
|-----|---------|------|
| `null` 值 | 填充为 0 | 所有整数特征的 null |
| `-1` 值 | 填充为 0 | 代表缺失的标记值 |
| `<= 0` 值 | 统一为 0 | 确保填充值统一 |
| 空列表 | 填充全 0 数组 | 变长特征为空时 |

### 越界异常值处理策略

| 配置 | 处理方式 | 说明 |
|-----|---------|------|
| `clip_vocab=True` | 裁剪为 0 | 超过 vocab_size 的值设为 0 |
| `clip_vocab=False` | 报错抛出 | 触发 `ValueError` |
| `vocab_size == 0` | 强制为 0 | 特征无条件设为 0，保证模型安全 |

### 越界检测与统计

- 统计信息包含：
  - 越界发生次数
  - 越界最大值
  - 越界最小值
  - 对应 vocab_size
- 可通过 `dump_oob_stats()` 导出到文件或日志
