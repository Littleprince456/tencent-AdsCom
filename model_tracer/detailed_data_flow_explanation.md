## 🎯 2. 阶段1: NS Token 生成

### 2.1 什么是 NS Token？
NS (Neural Schema) Token 是把用户和物品的各种特征压缩成的一组固定维度的向量，用于后续的序列交互！

### 2.2 处理步骤

#### 步骤 2.2.1 用户 NS Token (user_ns)
- **输入**: `user_int_feats` (形状 `(1, 90)`)
- **方法**: 按分组聚合 (使用 ns_groups.json)
- **分组配置**: 7 个分组：`U1~U7`
- **输出形状**: `(1, 7, 64)` (每个分组生成一个 64 维向量)

#### 步骤 2.2.2 物品 NS Token (item_ns)
- **输入**: `item_int_feats` (形状 `(1, 15)`)
- **分组配置**: 4 个分组：`I1~I4`
- **输出形状**: `(1, 4, 64)`

#### 步骤 2.2.3 用户稠密特征 Token (user_dense_tok)
- **输入**: `user_dense_feats` (形状 `(1, 623)`)
- **方法**: 通过线性层投影
- **输出形状**: `(1, 1, 64)` (单个 Token)

#### 步骤 2.2.4 合并所有 NS Token
- **方法**: concatenate 拼接
- **输出**: `ns_tokens` (形状 `(1, 12, 64)`：7+4+1)

## 🧠 5. 阶段4: MultiSeqHyFormerBlock (2 层)

这是模型的核心！把查询和 NS Token 放在一起做交互！

### 5.1 输入
- 查询：4 个序列查询，各 `(1, 1, 64)`
- NS Token：`ns_tokens` `(1, 12, 64)`

### 5.2 处理流程 (1个Block)
```
查询 ──┐
       ├─→ [交叉注意力] → [前馈网络] → 更新查询
NS Token ──┘
```

- 第 0 层 Block：处理初始的查询和 NS Token
- 第 1 层 Block：处理第 0 层输出的查询和 NS Token

这段有问题，实际上模型的结构是：
- 输入：
NS token：这是全局的，涵盖了 user 和 item 的静态特征。
query：这是每个序列生成的查询向量，是上一个组件MultiSeqQueryGenerator的输出。每个序列可能生成多个查询向量（这里只有 1 个），目前有 4 个序列，则意味着这部分的输入是 4 组查询向量，每组中有 1 个或者多个。
seq_list：这是每个原始序列，形状为 `(1, 256, 64)`。有 4 个序列。

- 结构：
有两层 block，在每个 block 内部，需要按顺序经过：
       - 原始序列过 transformer 编码（4 个序列就是 4 个 transformer 编码器）（这里不一定是经典 transformer，代码中定义了 3 种不同的 类 transformer结构）
       - 在各序列内部，query 和 编码后的 seq_list 进行交叉注意力，得到综合了 domain 内部信息的新的查询向量。
       - 拼接所有查询向量和 NS Token，得到最终的查询向量，在 dim=1 维度拼接
       - rankmixer 以及拆分回多个 query 向量和一个 ns token

```
MultiSeqHyFormerBlock
├── seq_encoders   [S个]  ← 每个序列一个编码器
├── cross_attns    [S个]  ← 每个序列一个交叉注意力
└── mixer          [1个]  ← 一个 RankMixer 做融合
```

## 📈 6. 阶段5: 输出投影

### 6.1 处理步骤

#### 步骤 6.1.1 拼接所有查询
- **输入**: 4 个序列查询，各 `(1, 1, 64)`
- **操作**: concatenate
- **输出**: `all_q_concat` (形状 `(1, 4, 64)`)

#### 步骤 6.1.2 展平
- **输入**: `all_q_concat` (形状 `(1, 4, 64)`)
- **操作**: 展平
- **输出**: `output` (形状 `(1, 256)`：4*64)

#### 步骤 6.1.3 投影到隐藏维度
- **输入**: `output` (形状 `(1, 256)`)
- **操作**: Linear(256 → 64)
- **输出**: `output_proj` (形状 `(1, 64)`)

#### 步骤 6.1.4 最终预测
- **输入**: `output_proj` (形状 `(1, 64)`)
- **操作**: Linear(64 → 1)
- **输出**: `logits` (形状 `(1, 1)`)

## 💡 总结

数据从原始的 parquet 表格（混合了整数、浮点数、列表）开始，经过以下过程：
1. **预处理**: 特征清洗、填充、对齐到固定长度
2. **Token化**: 把用户/物品特征压缩成 NS Token，把序列特征做 Embedding
3. **交互**: 通过 MultiSeqHyFormer 让查询和 NS Token 交互
4. **预测**: 把交互后的结果投影成最终的 logit 输出
