# 本文档中
本文档用来整理比赛使用的机器学习平台的使用方法，包括训练和评估两个阶段。
官方的使用手册：https://doc.weixin.qq.com/doc/w3_ALQATQZyACgCNqA7rGAELTHOn2rc5?scode=AJEAIQdfAAo65iNyGaALQATQZyACg

本文档是对该手册的简要整理。

# 概述
平台只提供训练和评估的算力支持，并不是一个用户友好的 IDE，因此所有的数据和脚本需要在本地调试好，然后传到平台上。在平台上启动`job`，会自动运行提交的脚本，开始训练或者评估任务。

# 训练
1. 
在平台上提交一个`training task`，填入任务信息，并且从本地 upload 脚本，提交即创建好了任务。

**注意**：upload的脚本中一定要有一个名称写死的 shell文件：`run.sh`，它是训练的总入口。运行任务后，平台会运行`bash run.sh`来启动训练。

2. 
提交任务后，可以点击`run`来运行任务，之后可以点击`output`或者`logs`查看详细日志。

3. 
模型训练过程中会保存有 checkpoint，如果想发布这个 ckpt，可以点击对应训练任务的`output`按钮，找到这个 ckpt 并选择提交，之后可以在侧边的`model management`中找到并且管理这个模型。平台最多保存 20 个发布后的模型。

4. 重要的环境变量
平台提供了目录，对应路径的名称以环境变量的形式输入了环境中，用的时候只要在脚本中写：

```
import os
os.environ.get("----")
```

几个环境变量：
- USER_CACHE_PATH
用于储存一些东西
- TRAIN_DATA_PATH
放训练数据
- TRAIN_CKPT_PATH
放模型的 checkpoint，而且只能存放在这个路径下。ckpt的命名规范：必须以`global_step`开头，后缀可以跟其他的东西
- TRAIN_TF_EVENTS_PATH
放训练时的一些图标


# 评估
1. 
在`model management`中找到想要评估的模型，点击选择`model evaluation`，点击`create evaluation`。

2. 
评估也需要脚本，这个脚本要从本地上传。同样地，上传的脚本需要包含一个名称写死的 infer.py 文件，这个文件中需要有一个 main() 函数，而且该函数没有参数。

**注意**：每 24 小时只能运行 3 次评估（失败的不算），每次评估时长最多 30 分钟。

3. 重要的环境变量
- USER_CACHE_PATH
用于储存一些东西，和训练的同目录
- MODEL_OUTPUT_PATH

- EVAL_DATA_PATH
放评估数据
- EVAL_RESULT_PATH
放评估的结果。尤其是预测结果要命名为`predictions.json`，格式为`{"user_id":"pred_prob"}`
- EVAL_INFER_PATH
放用于评估的脚本，即 `infer.py`
